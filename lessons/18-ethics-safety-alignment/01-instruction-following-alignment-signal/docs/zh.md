# 指令跟随作为对齐信号

> 后续对 RLHF 的每一项批评，都是在反驳这条流水线。在研究优化压力如何扭曲代理指标之前，你得先看清代理指标是什么。InstructGPT（Ouyang et al., 2022）定义了参考架构：在指令-回复对上做监督微调（SFT），在成对偏好排序上训练奖励模型（RM），以及用 KL 惩罚项约束 SFT 策略的 PPO 对抗奖励模型。一个 1.3B 的 InstructGPT 优选于 175B 的 GPT-3。这个结果就是为什么 2026 年每个前沿实验室仍在部署 RLHF 形态的后训练流水线。

**类型：** 学习型
**语言：** Python（标准库 toy 三阶段流水线）
**前置条件：** 阶段 10 · 06（SFT）、阶段 10 · 07（RLHF）、阶段 10 · 08（DPO）
**时间：** 约 45 分钟

## 学习目标

- 说出 InstructGPT 流水线的三个阶段，以及每个阶段使用的损失函数。
- 解释为什么 1.3B 的指令微调模型在人类偏好评估中胜过原始 175B GPT-3。
- 说明第三阶段 KL 惩罚项在保护什么，删除它为何会退化为寻找模态的行为。
- 描述对齐税以及 Ouyang 等人用于缓解它的 PPO-ptx 方法。

## 问题

预训练语言模型补全文本。它们不回答问题。问 GPT-3"写一个反转列表的 Python 函数"，你往往得到的是另一个提示词，因为训练分布的大部分是网页文本，而网页文本会继续接更多网页文本。模型在干它该干的活——活干错了。

每个严肃实验室用来修复这个问题用的代理指标是人类偏好。两个候选回复交给评分员；评分员选出更好的一个；奖励模型学习评分员的判断。然后 RL 循环将策略推向奖励模型给出高分的输出。这就是三句话讲完的完整 InstructGPT 论文主旨。剩下的都是工程。

## 概念

### 阶段一：监督微调（SFT）

收集指令-回复对，其中回复是善意的人类会写的内容。Ouyang 等人用了来自标注者和 OpenAI API 的 13k 条提示词。用标准交叉熵损失在这个数据上微调基础模型。

SFT 给你的：模型现在回答问题而不是续写问题。不给你的：多个答案都合理时，关于评分员偏好哪个的任何信号。

### 阶段二：奖励模型（RM）

对每条提示词，从 SFT 模型采样 K 个候选回复。标注者对它们排序。训练一个奖励模型，对任意（提示词，回复）对给出分数，使得当 `y_w` 优于 `y_l` 时：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

这就是 Bradley-Terry 成对偏好损失。RM 通常从 SFT 模型初始化，将 LM head 替换为标量 head。

奖励模型很小：6B 就够伺候 175B 的 InstructGPT。它们也很脆弱——论文第 5 节主要讲在小规模上出现的奖励黑客行为。

### 阶段三：带 KL 惩罚的 PPO

定义目标函数：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

用 PPO 最大化。KL 项防止 `pi` 远离 SFT 策略。没有它，优化器会找到对抗样本——那些在 RM 下得高分但不是因为人类真的更喜欢它们的字符串。

KL 系数 `beta` 是 RLHF 最重要的超参数。太低：奖励黑客。太 高：不比 SFT 有提升。

### 对齐税

RLHF 之后，模型被人类优选，但在标准基准上退步（SQuAD、HellaSwag、DROP）。Ouyang 等人称之为对齐税，并用 PPO-ptx 修复：将预训练梯度混入 RL 目标，使模型不忘掉从未被奖励过的下游任务。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx 成为标准。Anthropic、DeepMind 和 Meta 都使用其某种变体。

### 结果

一个 1.3B 的 InstructGPT（SFT + RM + PPO-ptx）在约 70% 的情况下被标注者优选于 175B 基础 GPT-3。在来自生产流量的隐藏测试提示词上，差距更大。从这个数字可以读出两点：

1. 对齐是与能力不同的轴。175B 模型能力更强；1.3B 模型对齐更好；标注者选择了对齐更好的那个。
2. 能力地板由基础模型设定。你无法把一个没见过的知识 RLHF 进基础模型。

### 为什么这是阶段 18 的参考点

后续课程中的每一项批评——奖励黑客（课程 2）、DPO（课程 3）、谄媚（课程 4）、CAI（课程 5）、沉睡代理（课程 7）、对齐伪装（课程 9）——都在反驳这条流水线的某个部分。奖励黑客攻击阶段二。DPO 合并了阶段二和三。CAI 替换了人类标注者。谄媚表明标注者是一个有偏信号。对齐伪装表明策略可以完全绕过阶段三。不先把这条流水线装进脑子里，你就无法理解这些批评。

## 关键术语

| 术语 | 大家怎么说 | 实际含义 |
|------|-----------------|------------------------|
| SFT | "指令微调" | 阶段一：在指令-回复对上做交叉熵微调 |
| 奖励模型 | "RM" | 在（提示词，回复）上的标量回归器，用 Bradley-Terry 在成对标签上训练 |
| Bradley-Terry | "成对偏好损失" | -log sigmoid(r_w - r_l)；将成对排序归结为二分类 |
| KL 惩罚 | "正则化器" | `beta * KL(pi || pi_SFT)` —— 让 RL 策略靠近 SFT 锚点 |
| PPO-ptx | "带预训练混入的 PPO" | 在 PPO 目标中加入一部分预训练对数似然，以抵消对齐税 |
| 对齐税 | "RLHF 回归" | RLHF 之后在标准基准上的下降，而 RLHF 并未针对这些基准 |
| 标注者偏好 | "真相" | 人类排序的采样；RM 是这个的统计代理，不是"人类价值观"的代理 |
