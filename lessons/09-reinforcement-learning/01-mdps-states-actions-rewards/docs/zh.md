# 马尔可夫决策过程：状态、动作与奖励

> 马尔可夫决策过程由五个要素构成：状态、动作、转移、奖励、折扣。强化学习中的所有算法——Q-learning、PPO、DPO、GRPO——都是在这个框架上进行优化。学透它一次，剩下的强化学习内容就能免费阅读。

**类型：** 学习型
**语言：** Python
**前置条件：** 阶段 1 · 06（概率与分布）、阶段 2 · 01（机器学习分类）
**时间：** 约 45 分钟

## 问题

你在写一个棋类 AI。或者一个库存规划器。或者一个交易智能体。或者训练推理模型的 PPO 循环。四个不同的领域，一个令人惊讶的事实：它们都能归纳为同一个数学对象。

监督学习给你 `(x, y)` 对，让你拟合一个函数。强化学习不给你标签——只有一系列状态、你采取的动作，以及一个标量奖励。这一步棋有没有赢？这次补货决策有没有省钱？这笔交易有没有盈利？这个 LLM 刚生成的 token 是否带来了更高的评判奖励？

在你形式化这个过程之前，你无法从中学习。"我看到了什么""我做了什么""接下来发生了什么""这有多好"——每一个都需要成为一个可以推理的对象。这个形式化就是马尔可夫决策过程。本阶段中的每一个 RL 算法，包括最后的 RLHF 和 GRPO 循环，都是在这个框架上进行优化。

## 概念

![马尔可夫决策过程：状态、动作、转移、奖励、折扣](../assets/mdp.svg)

**五个核心对象。**

- **状态** `S`。智能体做决策所需的全部信息。在 GridWorld 中是格子，在棋类中是棋盘，在 LLM 中是上下文窗口加上任何记忆。
- **动作** `A`。可供选择的选项。向上/下/左/右移动。走一步棋。输出一个 token。
- **转移** `P(s' | s, a)`。给定状态 `s` 和动作 `a`，下一个状态的分布。在棋类中是确定性的，在库存管理中是随机的，在 LLM 解码中几乎是确定性的。
- **奖励** `R(s, a, s')`。标量信号。赢了 = +1，输了 = -1。收入减去成本。GRPO 中的对数似然比项。
- **折扣** `γ ∈ [0, 1)`。未来奖励相对于当前奖励的权重。`γ = 0.99` 相当于约 100 步的视野；`γ = 0.9` 相当于约 10 步。

**马尔可夫性质** `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, …, s_t, a_t)`。未来只依赖于当前状态。如果不满足这个性质，说明状态表示不完整——这不是方法的问题，而是状态的问题。

**策略与回报。** 策略 `π(a | s)` 将状态映射到动作分布。回报 `G_t = r_t + γ r_{t+1} + γ² r_{t+2} + …` 是未来奖励的折扣总和。价值 `V^π(s) = E[G_t | s_t = s]` 是在策略 `π` 下从状态 `s` 开始的期望回报。Q 值 `Q^π(s, a) = E[G_t | s_t = s, a_t = a]` 是在从状态 `s` 开始并执行特定动作 `a` 时的期望回报。每一个 RL 算法都是估计这两个量之一，然后据此改进 `π`。

**贝尔曼方程。** 本阶段所有算法都在使用的不动点方程：

`V^π(s) = Σ_a π(a|s) Σ_{s', r} P(s', r | s, a) [r + γ V^π(s')]`
`Q^π(s, a) = Σ_{s', r} P(s', r | s, a) [r + γ Σ_{a'} π(a'|s') Q^π(s', a')]`

这些方程将期望回报分解为"这一步的奖励"加上"你到达位置的折扣价值"。递归的。本阶段中的每一个算法要么迭代这个方程直至收敛（动态规划），要么从中采样（蒙特卡洛），要么自举一步（时序差分）。

## 动手实现

### 第 1 步：一个极简确定性 MDP

一个 4×4 的 GridWorld。智能体从左上角开始，终点在右下角，每步奖励 -1，动作集为 `{up, down, left, right}`。见 `code/main.py`。

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    r, c = state
    nr = min(max(r + dr, 0), GRID - 1)
    nc = min(max(c + dc, 0), GRID - 1)
    return (nr, nc), -1.0, (nr, nc) == TERMINAL
```

五行代码。这就是整个环境。确定性转移、固定步惩罚、吸收性终止状态。

### 第 2 步：执行一个策略

策略是从状态到动作分布的函数。最简单的：均匀随机。

```python
def uniform_policy(state):
    return {a: 0.25 for a in ACTIONS}

def rollout(policy, max_steps=200):
    s, total, steps = (0, 0), 0.0, 0
    for _ in range(max_steps):
        a = sample(policy(s))
        s, r, done = step(s, a)
        total += r
        steps += 1
        if done:
            break
    return total, steps
```

运行随机策略 1000 次。平均回报约为 -60 到 -80。最优回报是 -6（沿对角线向右下的直线路径）。缩小这个差距就是阶段 9 的全部内容。

### 第 3 步：通过贝尔曼方程精确计算 `V^π`

对于小型 MDP，贝尔曼方程是一个线性系统。枚举所有状态，应用期望，反复迭代直到值不再变化。

```python
def policy_evaluation(policy, gamma=0.99, tol=1e-6):
    V = {s: 0.0 for s in all_states()}
    while True:
        delta = 0.0
        for s in all_states():
            if s == TERMINAL:
                continue
            v = 0.0
            for a, pi_a in policy(s).items():
                s_next, r, _ = step(s, a)
                v += pi_a * (r + gamma * V[s_next])
            delta = max(delta, abs(v - V[s]))
            V[s] = v
        if delta < tol:
            return V
```

这是迭代式策略评估。它是 Sutton & Barto 中的第一个算法，也是后续所有 RL 方法的理论基础。

### 第 4 步：`γ` 是一个有物理意义的超参数

有效视野大致为 `1 / (1 - γ)`。`γ = 0.9` → 10 步。`γ = 0.99` → 100 步。`γ = 0.999` → 1000 步。

太低会让智能体短视。太 高会导致信用分配变得嘈杂，因为很多早期步骤共同分享了远期奖励的责任。LLM 的 RLHF 通常使用 `γ = 1`，因为回合很短且有界。控制任务使用 `0.95–0.99`。长视野策略游戏使用 `0.999`。

## 陷阱

- **非马尔可夫状态。** 如果你需要最近三次观测才能做决策，那么"状态"就不仅仅是当前观测。修复方法：堆叠帧（DQN 在 Atari 上堆叠 4 帧）或使用循环状态（LSTM/GRU 处理观测序列）。
- **稀疏奖励。** 在大状态空间中，仅有最终胜利奖励几乎不可能学习。塑造奖励（中间信号）或用模仿学习引导（阶段 9 · 09）。
- **奖励黑客。** 优化代理奖励往往产生病态行为。OpenAI 的摩托艇竞赛智能体为了永远收集能量道具而原地打转，而不是完成比赛。始终根据目标结果定义奖励，而不是代理目标。
- **折扣设置错误。** 在无限视野任务上设置 `γ = 1` 会让每个价值都趋于无穷。要么用有限视野限制，要么设置 `γ < 1`。
- **奖励尺度。** 奖励 `{+100, -100}` 与 `{+1, -1}` 给出相同的最优策略，但梯度大小完全不同。在输入 PPO/DQN 之前，归一化到 `[-1, 1]` 左右。

## 实际使用

2026 年的技术栈将每一个 RL 流程在接触代码之前先归结为 MDP：

| 场景 | 状态 | 动作 | 奖励 | γ |
|-----------|-------|--------|--------|---|
| 控制（ locomotion、manipulation） | 关节角度 + 速度 | 连续力矩 | 任务相关的塑造奖励 | 0.99 |
| 游戏（棋类、围棋、扑克） | 棋盘 + 历史 | 合法走法 | 赢=+1 / 输=-1 | 1.0（有限） |
| 库存/定价 | 库存 + 需求 | 订货量 | 收入 - 成本 | 0.95 |
| LLM 的 RLHF | 上下文 token | 下一个 token | 结束时奖励模型得分 | 1.0（回合约 200 token） |
| 推理的 GRPO | 提示 + 部分响应 | 下一个 token | 结束时验证器 0/1 | 1.0 |

在写任何训练循环之前先写出这五个元组。大多数"RL 不 work"的问题报告都可以追溯到 MDP 公式化在纸上就已经出了问题。

## 练习

1. **简单。** 在 `code/main.py` 中实现 4×4 GridWorld 和随机策略 rollout。运行 10,000 个回合。报告回报的均值和标准差。与最优回报 (-6) 比较。
2. **中等。** 对均匀随机策略运行 `policy_evaluation`，`γ ∈ {0.5, 0.9, 0.99}`。将 `V` 作为 4×4 网格打印。解释为什么靠近终止点的状态价值随 `γ` 增大而增长更快。
3. **困难。** 将 GridWorld 随机化：每个动作以概率 `p = 0.1` 滑向相邻方向。重新评估均匀策略。`V[start]` 变好还是变差？为什么？

## 关键术语

| 术语 | 大家怎么说的 | 实际含义 |
|------|-----------------|-----------------------|
| MDP | "强化学习框架" | 满足马尔可夫性质的元组 `(S, A, P, R, γ)`。 |
| 状态 | "智能体看到的东西" | 在选定的策略类下对未来动态的充分统计量。 |
| 策略 | "智能体的行为" | 条件分布 `π(a | s)` 或确定性映射 `s → a`。 |
| 回报 | "总奖励" | 从当前步开始的折扣奖励总和 `Σ γ^t r_t`。 |
| 价值 | "一个状态有多好" | 在策略 `π` 下从 `s` 开始的期望回报。 |
| Q 值 | "一个动作有多好" | 在策略 `π` 下从 `s` 开始并执行动作 `a` 的期望回报。 |
| 贝尔曼方程 | "动态规划递归" | 将价值/Q 分解为一步奖励加上折扣后续价值的不动点分解。 |
| 折扣 `γ` | "未来与现在" | 远期奖励的几何权重；有效视野 `~1/(1-γ)`。 |
