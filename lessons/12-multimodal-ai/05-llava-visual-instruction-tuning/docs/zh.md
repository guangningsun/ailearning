# LLaVA 与视觉指令微调

> LLaVA（2023 年 4 月）是地球上被复制最多的多模态架构。它用 2 层 MLP 取代了 BLIP-2 的 Q-Former，用朴素的 token 拼接取代了 Flamingo 的门控交叉注意力，并在 15.8 万个由 GPT-4 从纯文本描述生成视觉指令回合上训练。2023 年至 2026 年间构建 VLM 的每个从业者都构建了 LLaVA 的某个变体。LLaVA-1.5 添加了 AnyRes。LLaVA-NeXT 提升了分辨率。LLaVA-OneVision 在一个配方中统一了图像、多图像和视频。本节课阅读配方，实现投影器，并解释为什么"更简单赢了"。

**类型：** 构建型
**语言：** Python（标准库，投影器 + 指令模板构建器）
**前置条件：** 阶段 12 · 02（CLIP）、阶段 11（LLM 工程——指令微调）
**时间：** 约 180 分钟

## 学习目标

- 构建一个将 ViT 块嵌入（维度 1024）映射到 LLM 嵌入维度（维度 4096）的 2 层 MLP 投影器。
- 走一遍 LLaVA 两阶段配方：(1) 在 55.8 万描述对上投影器对齐，(2) 在 15.8 万 GPT-4 生成的回合上进行视觉指令微调。
- 构建 LLaVA 格式提示，包含图像 token 占位符、系统提示和用户/助手回合。
- 解释社区为何从 Q-Former 转向 MLP，尽管 Q-Former 在 token 预算上更胜一筹。

## 问题

BLIP-2 的 Q-Former（第 12.03 课）将图像压缩到 32 个 token。简洁、高效、基准测试表现好。但它有两个问题。

首先，Q-Former 可训练，但其损失不是最终任务。阶段 1 训练 ITC+ITM+ITG。阶段 2 训练 LM 损失。查询学习某种中间表示，LLM 然后必须解码它。信息在瓶颈中丢失。

其次，Q-Former 需要 1.88 亿参数，在 LLaVA 2023 年的规模下，你必须与目标 LLM 共同设计它。更改 LLM，就要重新训练 Q-Former。更改视觉编码器，也要重新训练。每种组合都是独立的研发项目。

LLaVA 的答案简单得令人尴尬：取 ViT 的 576 个块 token，每个通过 2 层 MLP（`1024 → 4096 → 4096`），然后将所有 576 个送入 LLM 的输入序列。没有瓶颈。没有阶段 1 预训练于奇怪的目标。只是用直接的 LM 损失训练 MLP。

数据从哪里来？LLaVA 的第二个洞察：用 GPT-4（纯文本）生成指令数据。将图像的 COCO 描述和边界框数据发送给 GPT-4，要求它生成对话、描述和复杂推理问题。15.8 万个免费指令-响应回合。无需人工标注。

结果：一个能在 8 块 A100 上运行一天、在 MMMU 上击败 Flamingo、并且发布社区可扩展的开源检查点的 VLM。到 2023 年底，它已产生 50+ 个分支。

## 概念

### 架构

LLaVA-1.5（13B）：
- 视觉编码器：CLIP ViT-L/14 @ 336（阶段 1 冻结，阶段 2 可选解冻）。
- 投影器：带 GELU 激活的 2 层 MLP，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来是 Llama-3.1-8B）。

图像 + 文本提示的正向传递：

```
img -> ViT -> 576 个维度为 1024 的块
块 -> MLP -> 576 个维度为 4096 的 token
提示：系统 + "<image>" 占位符 + 用户问题
用 576 个投影 token 替换 <image> token
将完整序列送入 LLM
解码响应
```

图像占用 LLM 上下文的 576 个 token。在 2048 上下文中，剩 1472 个 token 用于文本。在 32k 上下文中，这只是一个舍入误差。

### 阶段 1：投影器对齐

冻结 ViT。冻结 LLM。只训练 2 层 MLP。数据集：55.8 万图像-描述对（LAION-CC-SBU）。损失：在投影图像 token 条件下对描述进行语言建模。

在批量 128 的单个 epoch 中，几小时就能完成。投影器学习将 ViT 空间映射到 LLM 空间。没有任务特定的监督。

### 阶段 2：视觉指令微调

解冻投影器（仍然可训练）。解冻 LLM（通常完全，有时 LoRA）。在 15.8 万视觉指令回合上训练。

指令数据是关键。Liu 等人生成方式：
1. 取一张 COCO 图像。
2. 提取文本描述（5 个人类描述 + 边界框列表）。
3. 用三个提示模板发送给 GPT-4：
   - 对话："生成关于这张图像的用户与助手之间的来回对话。"
   - 详细描述："对图像进行丰富、详细的描述。"
   - 复杂推理："问一个需要关于图像推理的问题，然后回答它。"
4. 将 GPT-4 的输出解析为（指令，响应）对。

这一切都不直接接触图像——只有文本描述。GPT-4 产生看似合理的图像内容。有一些噪音，但它有效：15.8 万回合足以解锁对话。

### 为什么社区复制了这个

- 没有阶段 1 特定损失需要调优。LM 损失贯穿始终。
- 投影器训练只需几小时，而不是几天。
- LLM 可以更换（LLaVA-Llama2、LLaVA-Mistral、LLaVA-Llama3），只需重新训练投影器。
- 视觉指令数据管道使用 GPT-4，为新领域重新生成成本低廉。

### LLaVA-1.5 和 LLaVA-NeXT

LLaVA-1.5（2023 年 10 月）添加：
- 学术任务数据（VQA、OKVQA、RefCOCO）混合到指令微调中。
- 更好的系统提示。
- 2048 → 32k 上下文。

LLaVA-NeXT（2024 年 1 月）添加：
- AnyRes：将高分辨率图像拆分为 2x2 或 1x3 的 336×336 裁剪网格，加上一张全局低分辨率缩略图。每个裁剪变成 576 个 token；每张图像总共约 2880 个视觉 token。OCR 和图表任务大幅提升。
- 更好的指令数据混合与 ShareGPT4V（高质量 GPT-4V 描述）。
- 更强的基础 LLM（Mistral-7B、Yi-34B）。

### LLaVA-OneVision

第 12.08 课深入介绍 OneVision。简短版本：相同的投影器，但训练课程涵盖单图像、多图像和视频在一个模型中共享视觉 token 预算。

### 与 Q-Former 的对比

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| 每张图像视觉 token | 32 | 576（基础）或 2880（AnyRes）|
| 可训练参数 | 1.88 亿 + LM | 4000 万 + LM |
| 阶段 1 损失 | ITC+ITM+ITG | 仅 LM |
| LLM 即插即用 | 需要重新训练 | 最小重新训练即可更换 |
| 多图像 | 笨拙 | 自然（拼接）|
| 视频 | 笨拙 | 自然（每帧拼接）|
| Token 预算 | 小 | 大 |

MLP 在简单性和 token 灵活性上胜出。Q-Former 在 token 预算上胜出。到 2023 年底，token 预算不再是约束（LLM 上下文增长到 32k-128k+），简单性占主导。

### 提示格式

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> 详细描述这张图像。 ASSISTANT: 图像显示...
```

`<image>` 是一个占位符 token。在 token 化之前，它被替换为 576 个视觉 token（或 AnyRes 的 2880 个）。token 化器看到的序列比训练时稍长，但 LLM 处理这个新输入，因为阶段 1 教会了它。

### 参数效率

LLaVA-1.5-7B 分解：
- CLIP ViT-L/14 @ 336：3.03 亿（阶段 1 冻结，阶段 2 通常解冻）。
- 投影器（2x 线性）：约 2200 万可训练。
- Llama-7B：70 亿。
- 总计：73 亿参数。阶段 2 可训练：完整 70 亿 + 2200 万投影器。

阶段 2 训练成本：8 块 A100 约 20 小时。这是关键数字——一天，一个节点，可复现。这就是 LLaVA 传播的原因。

## 使用它

`code/main.py` 实现：

1. 2 层 MLP 投影器（维度 16 → 32 → 32，用于玩具规模）纯 Python 实现。
2. 提示构建管道：系统提示 + `<image>` 替换为 N 个投影 token + 用户回合 + 助手生成占位符。
3. 576 token 视觉块在 LLM 上下文中的可视化（占 2k / 32k / 128k 上下文的百分比）。

## 交付它

本课产出 `outputs/skill-llava-vibes-eval.md`。给定一个 LLaVA 系列检查点，它运行 10 提示 vibes-eval 套件（3 个描述、3 个 VQA、2 个推理、2 个拒绝）并报告人类可读的评分卡。不是基准；是一个烟雾测试，确认投影器和 LLM 连接良好。

## 练习

1. 计算 2 层 MLP 投影器在 `1024 → 4096 → 4096` 时的可训练参数数量。有 GELU 和 bias 时，它占 LLaVA-13B 的多少比例？

2. 为"拒绝"案例构建一个 LLaVA 提示——图像包含私人个体。写出预期的助手响应。为什么 LLaVA 应该在零样本时拒绝？需要什么训练数据来强化拒绝？

3. 阅读 LLaVA-NeXT 博客的 AnyRes 部分。计算 1344×672 图像在 AnyRes 下的视觉 token 数量。与 336×336 基础 576 token 对比。

4. LLaVA 阶段 1 投影器用描述上的 LM 损失训练。如果你跳过阶段 1 直接进入阶段 2（视觉指令微调）会怎样？引用 Prismatic VLM 的消融（arXiv:2402.07865）回答。

5. LLaVA-Instruct-150k 使用 GPT-4 和 COCO 描述生成指令。对于新领域（医学 X 光、卫星图像），描述生成领域指令的四步数据管道。每步可能出什么问题？

## 关键术语

| 术语 | 大家怎么说的 | 实际含义 |
|------|----------------|------------------------|
| 投影器 | "MLP 桥接" | 带 GELU 的 2 层 MLP，将 ViT 维度映射到 LLM 维度 |
| 图像 token | "<image> 占位符" | 在推理前被 N 个投影视觉 token 替换的提示标记 |
| 视觉指令微调 | "LLaVA 阶段 2" | 在 GPT-4 生成的（图像，指令，响应）三元组上训练 |
| 阶段 1 对齐 | "投影器预训练" | 冻结 ViT 和 LLM，用描述上的 LM 损失训练投影器 |
| AnyRes | "多裁剪平铺" | 将高分辨率图像拆分为瓦片网格并连接每个瓦片的视觉 token |
| LLaVA-Instruct | "GPT-4 生成" | 15.8 万个由 COCO 描述 + GPT-4 合成的指令-响应对 |
| 视觉编码器冻结 | "骨干锁定" | CLIP 权重在阶段 1 不更新，有时阶段 2 也不更新 |
| ShareGPT4V | "更好的描述" | 由 GPT-4V 生成的 100 万密集描述，用于更高质量的对齐 |
| VQA | "视觉问答" | 回答关于图像的自由形式问题的任务 |
| Prismatic VLMs | "设计空间论文" | Karamcheti 2024 消融系统测试投影器和数据选择 |
