# Flamingo 与门控交叉注意力用于小样本视觉语言模型

> DeepMind 的 Flamingo（2022 年）在两方面领先于所有人。它展示了一个模型可以处理任意交错的图像、视频和文本序列。它还展示了视觉语言模型可以上下文学习——给几个样本提示（三个 (图像, 描述) 对），模型就能描述一张新图像，而无需任何梯度步。核心机制：门控交叉注意力层，插入到冻结 LLM 的已有层之间，带有一个学习到的 tanh 门，初始化为零以保留 LLM 的文本能力。本节课深入 Flamingo 的 Perceiver 重采样器和门控交叉注意力架构——这是 Gemini 交错输入和 Idefics2 视觉 token 的祖先。

**类型：** 学习型
**语言：** Python（标准库，门控交叉注意力 + Perceiver 重采样器演示）
**前置条件：** 阶段 12 · 03（BLIP-2 Q-Former）
**时间：** 约 120 分钟

## 学习目标

- 解释门控交叉注意力如何通过 tanh(gate) = 0 在初始化时保留冻结 LLM 的文本能力。
- 走一遍 Perceiver 重采样器：N 个图像块 → K 个固定的"潜在"查询，通过交叉注意力。
- 描述 Flamingo 如何处理交错的图像-文本序列，并使用考虑图像位置的因果掩码。
- 重现小样本多模态提示结构（3 个图像-描述示例后跟一个查询图像）。

## 问题

BLIP-2 将 32 个视觉 token 送入冻结 LLM 的输入层。适用于每次提示一张图像。但如果你想输入多张与文本交错的图像呢？比如"这是图像 A，描述它；这是图像 B，描述它；现在这是图像 C，描述它"？LLM 的自注意力需要在单一流中处理图像 token 和文本 token，哪些位置可以关注哪些图像的问题变得很麻烦。

Flamingo 的回答：完全不改变 LLM 的输入流。在已有 LLM 块之间插入额外的交叉注意力层。文本 token 照常通过 LLM 的因果自注意力流动。在每几个 LLM 块之间，文本 token 也通过一个新的门控层交叉关注图像特征。门（初始化为零）意味着在第零步，新层是空操作——模型行为完全像预训练的 LLM。随着训练进行，门打开，视觉信息开始流动。

Flamingo 回答的第二个问题：如何处理每个提示中可变数量的图像（0、1 或多张）？Perceiver 重采样器——一个小型交叉注意力模块，接收任意数量的块并产生固定数量的视觉潜在 token。无论提示中有多少图像，LLM 交叉注意力层看到的形状都是相同的。

## 概念

### 冻结的 LLM

Flamingo 从一个冻结的 Chinchilla 70B LLM 开始。所有 70B 权重保持不变。已有的文本自注意力和 FFN 正常运行。

### Perceiver 重采样器

对于提示中的每张图像，ViT 生成 N 个块 token。Perceiver 重采样器有 K 个固定的可学习潜在向量（Flamingo 使用 K=64）。每个重采样器块有两个子步骤：

1. 交叉注意力：K 个潜在向量关注 N 个块 token（Q 来自潜在向量，K/V 来自块）。
2. 潜在向量内部的自我注意力 + FFN。

经过 6 个重采样器块后，输出 K=64 个维度为 1024 的视觉 token，无论 ViT 产生了多少个块。224×224 图像（196 个块）和 480×480 图像（900 个块）都输出为 64 个重采样器 token。

对于视频，重采样器在时间维度上应用：每一帧的块产生 64 个潜在向量，时间位置编码让模型区分 t=0 和 t=N。整个视频变成 T × 64 个视觉 token。

### 门控交叉注意力

在冻结 LLM 的每 M 层之间（Flamingo 使用 M=4），插入一个新的门控交叉注意力块：

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha` 是一个初始化为零的可学习标量。
- `tanh(0) = 0`，所以在初始化时门控分支贡献为零。
- 当 `alpha` 偏离零时，交叉注意力的贡献平滑增长。
- 残差连接意味着即使门完全打开，也不会覆盖 LLM 的文本表示；只是在顶部添加视觉信息。

这是 Flamingo 中唯一最重要的设计选择：视觉条件化是加性的、门控的、初始化为零的。步长为 0 的 Flamingo 在纯文本输入上就是完美的 Chinchilla 70B。

### 交错输入的掩码交叉注意力

在类似 "<image A> 描述 A <image B> 描述 B <image C> ?" 这样的提示中，每个文本 token 应该只看到序列中出现在它之前的图像。交叉注意力掩码强制执行：位置 `t` 处的文本 token 只关注图像索引 `i < i_t` 的图像重采样器 token，其中 `i_t` 是位置 `t` 之前最接近的图像。"只看到最近的前一个图像"或"看到所有前一个图像"都是有效选择；Flamingo 选择前者。

### 上下文小样本学习

Flamingo 提示如下：

```
<image1> 一张猫的照片。 <image2> 一张狗的照片。 <image3> 一张...
```

模型看到补全模式并输出"鸟"（或者图像 3 显示的内容）。无需梯度步。冻结 LLM 的上下文学习能力通过门控交叉注意力传递——这是论文的核心观点，也是它重要的原因。

### 训练数据

Flamingo 在三个数据集上训练：

1. MultiModal MassiveWeb (M3W)：4300 万个包含交错图像和文本的网页，重建阅读顺序。
2. 图像-文本对（ALIGN + LTIP）：44 亿对。
3. 视频-文本对（VTP）：2700 万个短视频片段。

OBELICS（2023 年）是对交错网页语料库的开源复现，Idefics、Idefics2 和大多数开源的"Flamingo 风格"模型都在其上训练。

### OpenFlamingo 和 Otter

OpenFlamingo（2023 年）是开源复现。架构相同（Perceiver 重采样器 + 冻结 LLM 上的门控交叉注意力，基于 LLaMA 或 MPT）。检查点有 3B、4B、9B 版本。由于基础 LLM 较小且数据较少，质量落后于 Flamingo。

Otter（2023 年）在 OpenFlamingo 基础上构建，使用 MIMIC-IT（一个多模态指令数据集）进行指令微调，表明门控交叉注意力也适用于指令遵循。

### 后代

- Idefics / Idefics2 / Idefics3：Hugging Face 的门控交叉注意力传承，越来越简化（Idefics2 放弃了重采样器，转而使用带自适应池化的直接块 token）。
- Flamingo 到 Chameleon 的过渡：到 2024 年，许多团队转向早期融合（第 12.11 课）；Flamingo 风格的门控交叉注意力在需要冻结骨干网的生产环境中仍然保留。
- Gemini 的交错输入：概念上继承了 Flamingo 的交错格式灵活性，尽管确切机制是专有的。

### 与 BLIP-2 的对比

| | BLIP-2 | Flamingo |
|---|---|---|
| 视觉桥接 | 在输入处 Q-Former 一次 | 每 M 层门控交叉注意力 |
| 视觉 token | 每张图像 32 个 | 每张图像每交叉注意力层 64 个 |
| 冻结 LLM | 是 | 是 |
| 小样本上下文学习 | 弱 | 强——论文的核心 |
| 交错输入 | 无原生支持 | 是，设计目标 |
| 训练数据 | 1.3 亿对 | 13 亿对 + 4300 万交错网页 |
| 参数数量 | 1.88 亿训练 | 约 100 亿训练（交叉注意力层）|
| 计算量 | 8 块 A100 数天 | 数千块 TPUv4 数周 |

预算有限做单图像 VQA 选 BLIP-2。需要交错、小样本或多图像推理选 Flamingo/Idefics2。

## 使用它

`code/main.py` 演示：

1. 在 36 个假块 token 上使用 8 个可学习潜在向量的 Perceiver 重采样器（纯 Python 交叉注意力）。
2. 门控交叉注意力步骤，`alpha = 0` → 输出等于输入（LLM 不变），然后 `alpha = 2.0` → 视觉贡献混合进来。
3. 交错掩码构建器，为"(图像 1) (文本 1) (图像 2) (文本 2)"序列生成 2D 注意力掩码。

## 交付它

本课产出 `outputs/skill-gated-bridge-diagnostic.md`。给定一个开源 VLM 的配置（重采样器 Y/N、交叉注意力频率、门方案），它识别 Flamingo 血统元素并解释冻结策略。对于调试微调为何导致文本性能下降很有用（答案：门打开得太快太宽）。

## 练习

1. 计算 Flamingo-9B 的视觉参数数量：9B LLM + 14 亿门控交叉注意力层 + 6400 万重采样器。训练参数占总参数的比例是多少？

2. 在 PyTorch 中实现门控残差 `y = tanh(alpha) * cross + x`。通过实验证明 `alpha=0` 时，`y==x` 在初始化时完全成立。

3. 阅读 OpenFlamingo 第 3.2 节（arXiv:2308.01390），了解他们如何处理每个提示图像数量不同的批处理。描述填充策略。

4. 为什么 Flamingo 的交叉注意力掩码让文本 token 只关注最近的前一个图像，而不是所有前一个图像？阅读 Flamingo 论文第 2.4 节并解释权衡。

5. 上下文小样本：为一个新的 Flamingo 变体构建一个包含 4 个"图像 → 主物体颜色"示例的提示。描述当示例数量从 0 变化到 8 时期望的准确率模式。

## 关键术语

| 术语 | 大家怎么说的 | 实际含义 |
|------|----------------|------------------------|
| Perceiver 重采样器 | "固定潜在交叉注意力" | 从可变数量的输入块产生 K 个固定 token 的模块 |
| 门控交叉注意力 | "Tanh 门控桥接" | 残差层 `y = tanh(alpha)*cross + x`，可学习 alpha，初始化为 0 |
| 交错输入 | "混合序列" | 图像和文本按阅读顺序自由混合的提示格式 |
| 冻结 LLM | "无 LLM 梯度" | 文本 LLM 的权重不更新；只有重采样器 + 交叉注意力层训练 |
| 小样本 | "上下文示例" | 在提示中给出几个（图像，答案）对；模型无需微调就能泛化 |
| OBELICS | "交错网页语料库" | 1.41 亿个包含图像和文本的网页开源数据集，按阅读顺序排列 |
| Chinchilla | "70B 冻结基础" | Flamingo 的冻结文本 LLM，来自 DeepMind 的 Chinchilla 论文 |
| 门调度 | "alpha 如何移动" | 交叉注意力门在训练期间打开的速率 |
| 交叉注意力频率 | "每 M 层" | 门控交叉注意力块插入的频率；Flamingo 使用 M=4 |
| OpenFlamingo | "开源复现" | MosaicML/LAION 在 3-9B 的开源检查点；架构与 Flamingo 完全相同 |
