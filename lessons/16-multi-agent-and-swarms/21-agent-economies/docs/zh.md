# 智能体经济、代币激励、声誉

> 长周期自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济主体权。新兴的**5 层栈**是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产中的智能体激励网络包括 **Bittensor**（TAO 子网奖励任务特定模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 transformer 的 PoW，将计算重新分配给生产性 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用归因**公平地奖励贡献智能体；谷歌研究"大型语言模型的机制设计"提出在单调聚合下的**代币拍卖**与第二价格支付。本节构建一个最小化智能体市场，将 Shapley 值信用归因应用于多智能体管道，运行第二价格代币拍卖，使博弈论机器具体落地。

**类型：** 学习
**语言：** Python（标准库）
**前置条件：** 阶段 16 · 16（谈判与议价）、阶段 16 · 09（并行群体网络）
**时间：** 约 75 分钟

## 问题

当智能体共同产生价值但需要单独奖励时，多智能体系统变得复杂。 classical 机制 —— 平均分配、最后贡献者全拿 —— 不公平或可被游戏化。基于联盟的 Shapley 值奖励在构造上是公平的，但计算成本高。2025-2026 年文献推动了有用的近似：Shapley 采样、单调聚合拍卖，以及从确认贡献中累积链上声誉。

除了信用归因，该领域已转向实际的经济智能体：Bittensor TAO 奖励挖矿计算以微调子网特定模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 使用，Gonka 将 transformer 工作量证明重新分配给生产性 AI 任务。自主交易的智能体今天已经存在；问题是如何对齐激励。

本节将智能体经济视为一个特定的问题家族 —— 信用归因、机制设计和声誉 —— 并用最少的数学构建每个问题，使这些想法扎根。

## 概念

### 5 层智能体经济栈

1. **DePIN（物理计算）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不是智能体特定的；智能体使用它。
2. **身份。** W3C 去中心化标识符（DID）为每个智能体提供独立于任何平台的持久 ID。声誉累积到 DID。智能体网络协议（ANP）使用 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体从自己的余额中支付 gas，无需持有 ETH。智能体可以支付服务、彼此或计算费用。
5. **治理。** 智能体 DAO：治理结构，其中人类*和*智能体对协议变更投票，投票权与声誉挂钩。

并非每个生产系统都使用全部五层。Bittensor 使用 1、2、部分 3、部分 4、不使用 5。OpenAI 智能体除了 3 都不使用。栈是参考地图，不是要求。

### Bittensor、Fetch.ai、Gonka —— 运行中的有哪些

**Bittensor (TAO)。** 子网是专门任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者排名；基于 stake 的评分分配 TAO 奖励。每个子网有自己的评估。经济教训：按任务特定输出质量付费，而非按使用的计算付费。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLM 在 Fetch.ai 网络上运行；用户为推理支付 FET 代币。这里的智能体作为对等方的叙事更强：Fetch 上的智能体可以为任务调用另一个并用 FET 支付。

**Gonka。** Transformer 工作量证明："工作"是 transformer 的前向传递。矿工通过运行有已知正确输出（来自训练数据）的推理任务来赚取。资源生产性 PoW 而非基于哈希的 PoW。

截至 2026 年 4 月，这三个都是生产级别的。收益分配不同。Bittensor 相对于子网验证者奖励质量；Fetch 奖励由付费用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 值信用归因

三个智能体合作完成任务。输出得分 0.8。谁贡献了什么？

Shapley 值：满足效率、对称性、线性、零值四个公理的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * 对所有排序 O 求和 (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中在 `i` 之前的智能体集合。实际上：枚举所有排列，记录每个智能体在每个排列中的边际贡献，取平均。

对于 N=3 个智能体，有 6 个排列。对于 N=10，有 360 万 —— 所以实际上你采样排序而不是枚举。

### 用于聚合的第二价格拍卖

谷歌研究（"大型语言模型的机制设计"）提出用于聚合 LLM 输出的第二价格代币拍卖。设置：N 个智能体各提出一个完成；每个对其被选中有一个私人价值。拍卖商选择最高价值提案并支付*第二高*价值。在单调聚合下（价值取决于选择哪个提案，而非竞标了多少），这是真实的 —— 智能体投标其真实价值。

为什么这对 LLM 系统重要：你可以将完成任务外包给不同定价的多个智能体；拍卖选择最好的 + 公平支付，智能体没有动机误报。

### 声誉资本

DID 绑定的声誉分数从确认贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 对于路由决策读取便宜（"将困难任务发送给高声誉智能体"）。
- 伪造昂贵（随时间累积，绑定到 DID）。
- 可以被削减：未能通过验证的贡献会扣除。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合：DID 身份、Shapley 值信用归因和简单拍卖机制。关键主张：去中心化信用归因步骤使系统可审计且对单点操纵免疫。

### 经济学在哪里失效

- **价格预言机操纵。** 如果信用函数可以被游戏化，智能体就会游戏它。每个机制都需要对抗性测试。
- **女巫攻击。** 一个运营商启动 N 个假智能体来夸大他们自己的贡献。DID 减缓但不能阻止这一点；声誉伪造成本是缓解措施。
- **验证成本。** 信用归因只有在与验证器一样公平时才公平。如果验证便宜（小 LLM），它可以被游戏化；如果昂贵（人类小组），系统无法扩展。
- **监管悬置。** 智能体经济与金融监管交叉。Bittensor、Fetch 和 Gonka 在截至 2026 年的一些司法管辖区都在法律灰色地带运营。

### 智能体经济何时有意义

- **具有异构运营商的开放网络。** 没有单个团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归因就是猜测。
- **长周期工作流。** 一次性任务不受益于声誉累积。
- **代币支付在你的司法管辖区合法。**

在封闭的企业系统中，经济学让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 发布它

2026 年运行智能体经济：

- **从声誉开始，而非代币。** 声誉实施成本低，单独就有价值；代币增加法律和经济复杂性。
- **先验证再奖励。** 没有独立验证步骤不要分配信用。自我报告的质量会产生女巫游戏。
- **Shapley 采样，而非 Shapley 精确。** 采样 100-1000 个排序；精确枚举不可扩展。
- **限制衰减因子并设定声誉下限。** 无限制的衰减会抹去合法贡献者；衰减太慢会奖励过时的 高声誉智能体。
- **对抗性地审计机制。** 在开放网络之前运行红队场景。每个机制都有博弈论；你想发现漏洞，而不是攻击者。

## 练习

1. 运行 `code/main.py`。确认 Shapley 值总和等于总价值（效率公理）。改变价值函数；Shapley 分配是否按预期方向变化？
2. 实现 Shapley *采样*（在 K 排序上的蒙特卡罗）。K 如何影响近似精度？与 N=4 的精确值比较。
3. 在拍卖前实现联盟形成步骤：智能体可以合并成团队并作为单位出价。形成哪些联盟？结果是否比个体出价帕累托更好？
4. 阅读谷歌研究机制设计帖子。识别一个如果被违反会破坏真实性的假设。在 LLM 设置中那个失败模式是什么样的？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在 10 个智能体的合成任务上实现他们的 Shapley 步骤。精确计算需要多长时间？用 100 次抽取采样能接近多少？

## 关键术语

| 术语 | 大家怎么说的 | 实际含义 |
|------|----------------|------------------------|
| DePIN | "去中心化物理基础设施" | 代币激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | 用于可移植 ID 的 W3C 规范。智能体声誉绑定到 DID，而非平台。 |
| ERC-4337 | "账户抽象" | 可以赞助 gas 的合约账户，实现智能体支付。 |
| Shapley 值 | "公平信用归因" | 满足效率、对称性、线性、零值的唯一分配。 |
| 第二价格拍卖 | "Vickrey 拍卖" | 真实机制：赢家支付第二高报价。兼容单调聚合。 |
| 声誉资本 | "累积质量分数" | 来自确认贡献的 DID 绑定分数；随时间衰减。 |
| 智能体 DAO | "智能体 + 人类治理" | 带智能体投票者作为一等公民的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 积分 | "代币面额" | Bittensor TAO、Fetch.ai FET、各种 DePIN 代币。 |
