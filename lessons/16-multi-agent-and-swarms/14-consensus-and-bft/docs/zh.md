# 智能体的共识与拜占庭容错

> 经典的分布式系统 BFT 遇上随机 LLM。在 2025-2026 年，出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）用置信度探测来加权每个投票；**DecentLLMs**（arXiv:2507.14928）采用无领导者模式，并行工作节点提议加几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类，将节点分为 Core 和 Edge。《AI 智能体能达成一致吗？》（arXiv:2603.01213）的诚实经验结果是，即使是标量一致在今天也是脆弱的——一个欺骗性智能体就能危及混合智能体的共识。BFT 是必要的但不是充分的。本节课构建一个最小 BFT 协议，注入三种智能体特定攻击（拜占庭谎言、谄媚从众、相关错误同质化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 16 · 07（心智社会与辩论）、阶段 16 · 13（共享内存）
**时间：** 约 75 分钟

## 问题

你有 N 个 LLM 智能体，每个产生一个答案。它们意见不一致。多数投票选出了错误的那个，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的失效模式）。第三个智能体恰好以一种新奇的方式出错——所以多数是假的多数。

现在加上一个欺骗性智能体：它故意撒谎。或者一个谄媚性智能体：它同意最后发言的人。在经典 BFT 中，假设是拜占庭节点是 `f < n/3` 的比例，并且行为任意。2026 年的现实是，LLM 节点即使是诚实的也是随机的，在模型间相关，并受彼此输出的影响。你不能将它们视为独立的伯努利投票者。

经典 BFT（PBFT，1999 年）没有错——它是不完整的。它处理任意的位翻转。它不处理"三个诚实智能体因为共享训练数据而共享一个幻觉"。本节课从 PBFT 的基础上构建，并叠加三个 2025-2026 年的适配。

## 概念

### 经典 BFT 给你什么

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。该协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定证书）。在 `n >= 3f + 1` 个诚实或恶意节点中对单个值达成一致。

保证很强，但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点真的是诚实的。** 诚实输出的正确性不是问题；该协议只对齐分歧。
3. **问题有真实答案。** 对错误事实的共识仍然是共识。

LLM 智能体违反了这三条。两个运行相同基础模型的智能体共享故障。一个"诚实"的 LLM 仍然会产生幻觉。在模糊问题上，"真相"是智能体们决定的内容——没有外部预言机。

### 三种 LLM 特定攻击

**拜占庭谎言。** 一个智能体输出一个故意错误的答案。如果 `f < n/3`，经典 BFT 可以处理。

**谄媚从众。** 一个智能体在投票前读取其他人的答案，并与最后发言的人对齐。不是恶意的，但与最大的声音相关。经典 BFT 不能防止这种情况，因为该智能体通过了每个签名检查。

**相关错误同质化。** 三个智能体共享一个基础模型。它们产生相同的错误幻觉。多数是错误的。经典 BFT 没有帮助，因为所有三个都"诚实地"同意。

### 2025-2026 年的回应

**CP-WBFT**（arXiv:2511.10400）——置信度探测加权 BFT。每个投票者将其答案附加一个置信度探测（一个自我报告的概率，或一个独立校准模型的预测）。投票权重与置信度成比例缩放。报告在完整图上达到 +85.71% 的 BFT 改进。可缓解：谄媚从众（从众的智能体在其自愿立场上的置信度往往较低）。

**DecentLLMs**（arXiv:2507.14928）——无领导者。并行工作智能体提议，评估智能体对提议评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时具有鲁棒性。可缓解：拜占庭谎言和相关错误（几何中位数对异常值鲁棒，拉向密集聚类，而不是模型偏差的平均值）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史学习到的信任分数分配。将智能体聚类为 Core 和 Edge；Core 智能体必须首先达成共识，Edge 智能体跟随。可缓解：可扩展性（Core 共识小而快）和部分同质化（Core 可以为多样性选择）。

### 经验结果：《AI 智能体能达成一致吗？》（arXiv:2603.01213）

该论文测量了 LLM 智能体在多个前沿模型上的标量一致（LLM 智能体对一个单一数值的意见一致）。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准测试的标量问题上以超过 30% 的比率不同意。
- 一个采用欺骗人设的单一智能体可以将混合智能体共识拉离诚实基线 40 多个百分点。
- 不同意率与模型多样性相关——异构集成比同构集成不同意更多（好：错误不相关）但也漂移得更慢（坏：达成一致的时间更长）。

启示：BFT 给你对齐输出的机制，但不告诉你对齐的输出是否正确。结合验证（阶段 16 · 08 角色专业化）、多样性（阶段 16 · 15 辩论变体）和评估智能体（阶段 16 · 24 基准测试）。

### 核心协议，简化版

用于 LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i，范围 [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义聚类（等价答案）分组
5. 聚合器为每个聚类 C 计算权重：
     w(C) = sum_{i in C} c_i
6. 若 max > threshold * sum(c_i)，则 winner = 权重最大的聚类
   否则：重试或升级
7. 少数民族聚类记录溯源，供事后审计
```

语义聚类步骤是 LLM 特定的技巧。两句话"该研究报告了 4.2%"和"4.2% 的提升"是同一个聚类。朴素的字符串相等检查会漏掉这个。在生产中，使用便宜的嵌入模型或明确的规范化。

### 阈值调优

`threshold` 参数决定何时接受、何时重试。太低：你接受了弱多数。太 高：你永远不会接受任何东西。经验范围：`n=5-7` 时为 0.5-0.67，更小的 `n` 需要更高。低于阈值时，升级给人类或不同的智能体集合。

### 共识无济于事的地方

- **模糊问题。** 如果问题没有真实答案，共识是一种观点。称之为观点。
- **复合问题。** "写代码并解释它"——两个答案。分别对每个投票。
- **对抗性多轮。** 如果智能体可以观察先前的轮次并模仿（Du 2023 辩论），它们无论真假都开始彼此同意。限制轮次（通常 2-3 轮）。

## 构建

`code/main.py` 实现：

- `AgentVoter` — 带（答案，置信度）的脚本化策略。
- `MajorityVote` — 经典多数制。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 对评分提议的几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一条答案，置信度相同。
3. `monoculture`：三个智能体共享一个错误答案（相关错误），置信度适中。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）→ 最终答案的表格，正确答案高亮显示。多数制在同质化案例中失败。CPWBFT 的置信度加权缓解了谄媚从众。当同质化少于一半时，DecentLLMs 的几何中位数拉向诚实聚类。

## 使用

`outputs/skill-consensus-designer.md` 为多智能体集合设计共识协议：聚类方法、加权、阈值，以及低于阈值的轮次的升级策略。

## 交付

在交付任何共识机制之前：

- **用至少上述三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是静默失败。
- **记录每个少数民族聚类并附上溯源。** 少数民族聚类是你的相关错误早期预警系统。
- **强制限制轮次。** 不要"继续辩论直到达成一致"——那奖励谄媚从众。
- **将一致与正确分开。** 共识输出进入验证器；验证器独立于集合。
- **监控同意率。** 急剧上升意味着从众偏见；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认多数制在同质化攻击中失败，但当同质化置信度低于 0.7 时，CPWBFT 部分缓解了它。
2. 添加第四种攻击模式：**沉默弃权**——一个智能体拒绝回答（"我不知道"）。每个聚合器应该如何对待弃权？实现你的选择。
3. 将语义聚类从字符串规范化交换为嵌入相似度（使用任何开源嵌入模型）。谄媚从众攻击会发生什么变化？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（一个独立的校准模型检查每个智能体的自我报告置信度）。在同质化场景中测量准确度提升。
5. 阅读《AI 智能体能达成一致吗？》（arXiv:2603.01213）。复现一个简化的标量一致实验：三个智能体，一个标量问题，欺骗人设提示。CPWBFT 或 DecentLLMs 能捕捉到它吗？

## 关键术语

| 术语 | 大家怎么说的 | 实际含义 |
|------|----------------|------------------------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 年协议，用于在 `f < n/3` 个任意故障下达成共识。 |
| 拜占庭 | "任何不良行为" | 一个节点可以撒谎、丢弃消息、静默失败——除了安全崩溃之外的任何行为。 |
| 置信度探测 | "你有多确定？" | 附加到投票上的自我报告或校准器预测的概率。 |
| 语义聚类 | "相同答案，不同措辞" | 在计票前将等价答案分组。 |
| 几何中位数 | "鲁棒中心" | 最小化到样本点距离之和的点。不像均值那样对异常值敏感。 |
| 同质化 | "相同模型，相同故障" | 当智能体共享训练数据或基础模型时的相关错误。 |
| 谄媚从众 | "与大声者一致" | 智能体的投票偏向于首先/最大声发言的人。 |
| Core/Edge | "层次 BFT" | WBFT 分裂：先做小的 Core 共识，Edge 节点跟随。限制延迟。 |
